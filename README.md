# jakobhelton.github.io

This is the source repository for my personal website at [jakobhelton.github.io](https://jakobhelton.github.io). I am an astrophysicist and Evolving Universe Postdoctoral Fellow in the Department of Astronomy and Astrophysics at The Pennsylvania State University. The site covers my research, publications, presentations, blog, and art.

```bash
# 5. Enable GitHub Pages (Settings → Pages → Deploy from a branch → main, root)

# 6. Trigger the first data update (Actions → Daily Data Update → Run workflow)
```

## Design

I designed the site around a deep space dark theme (`#08090c` base) with a jade green (`#00a878`) accent color. Typography follows LaTeX conventions: Playfair Display for headings and Libre Franklin for body text, with fully justified alignment and no word hyphenation. The `text-wrap: pretty` property prevents orphan words (lone words on the last line of a paragraph), and `text-wrap: balance` is applied to headings. An animated star field with twinkling stars, faint diffuse nebulae, and spiral galaxy smudges runs on every page as a background canvas. Stars are rendered from a power-law size distribution (70% tiny, 20% medium, 9% bright, 1% brilliant), with ~27% drawn as colored pastels (soft blues, yellows, oranges, violets, pinks, gold) and ~10% rendered as ellipsoidal shapes. Each page load generates 4–8 nebulae (radial gradients, often with a secondary offset lobe) and several spiral galaxies with logarithmic arms and compact bright cores. Dark and light modes are both supported, with the user's preference persisted in `localStorage`. The layout is fully responsive, with hamburger navigation on mobile. All NASA/JWST images include credited attribution with a clickable source link.

## SEO and Crawlability

**`sitemap.xml`** is a standard XML file that lists every public URL on the site along with a `<changefreq>` hint and a `<priority>` score (0.0–1.0). Search engines like Google and Bing read it to discover pages they might otherwise miss — especially pages that are not linked from anywhere prominent. Without a sitemap, a crawler might never index a deeply nested blog post or an art gallery sub-page. When you add new pages (blog posts, presentations, art galleries), add a corresponding `<url>` entry to `sitemap.xml`.

**`robots.txt`** is a plain-text file served at the root of every domain. Crawlers fetch it before indexing anything else. It can restrict which paths a bot may visit or, as here, simply confirm that everything is allowed and point to the sitemap:

```
User-agent: *
Allow: /
Sitemap: https://jakobhelton.github.io/sitemap.xml
```

The `Sitemap:` directive tells every compliant crawler exactly where to find the sitemap without requiring manual submission to each search engine. Together, `sitemap.xml` and `robots.txt` are the minimum two files needed for reliable, complete search-engine indexing of a static site.

**Canonical links** (`<link rel="canonical" href="...">`) are present on every page, pointing to the page's own absolute URL. Canonical links prevent duplicate-content penalties in search engines when a page is accessible via multiple paths (e.g., with and without a trailing slash, or over HTTP vs HTTPS). Each canonical URL matches the corresponding `og:url` value on that page.

**JSON-LD structured data** (`<script type="application/ld+json">`) is embedded in the `<head>` of every page. It uses schema.org vocabulary to describe the site's content in a machine-readable format, enabling Google rich results and improving how the site appears in search engine results pages. The home page and biography page use `Person` / `ProfilePage` schemas; the publications page uses `CollectionPage`; blog posts use `BlogPosting`; art and other pages use `WebPage`.

## Social Sharing

Every page includes `og:image` and `twitter:image` meta tags pointing to `assets/images/og_image.png`, a 1200×630 dark-mode social card generated by `scripts/generate_og_image.py` using Pillow. The image features the site's deep space color palette, name, title, affiliation, and URL against a star field background. It is regenerated automatically as part of the daily pipeline.

## Performance

The landing page includes a `<link rel="preload" as="image" href="assets/images/hero_1.png" fetchpriority="high">` hint in the `<head>`, which instructs the browser to begin fetching the first hero image immediately during HTML parsing — before the CSS and JavaScript have loaded. This eliminates the largest contentful paint delay caused by the hero image being discovered late.

## Reading Time

Blog post pages display a dynamically computed reading time and word count alongside the static post metadata. `js/main.js` detects any page containing a `.bio-section` element, counts the words in that section (at 200 WPM), and replaces the static `~X min read` placeholder in the `.page-description` with a live value of the form `~X min read · Y words`. The blog listing page (`blog/index.html`) also shows a static `~X min read` estimate next to each card's date, giving readers a quick sense of length before clicking through.

## Print Stylesheet

`css/print.css` is loaded on every page with `media="print"`. It resets colors and backgrounds to black-on-white, hides navigation, star field, interactive controls, and back-links, collapses multi-column grids to a single column, and appends URLs after link text. The `@page` rule sets A4 margins. This makes any page on the site suitable for printing or saving as a PDF.

## Custom 404 Page

`404.html` is served automatically by GitHub Pages for any URL that does not match an existing file. It uses the same dark theme, navigation, star field, and footer as all other pages. The page displays the error code in accent green, a short message, and links back to Home, Publications, and Blog.

## Featured Publication

The landing page includes a static featured publication card below the scroll sections. It displays the figure (`assets/images/featured_publication.png`), title, first authors, journal reference, and full abstract. LaTeX math in the abstract is rendered client-side by [KaTeX](https://katex.org/) via `$...$` delimiters using the auto-render extension. To update the featured publication, edit the `#featured-pub` section in `index.html` and replace `assets/images/featured_publication.png`.

## Auto-Updating Pipeline

A GitHub Actions workflow (`.github/workflows/update_data.yml`) runs daily at 06:00 UTC and on manual dispatch, using `actions/checkout@v4` and `actions/setup-python@v5`. The pipeline executes five scripts in order.

**`fetch_publications.py`** queries the SciX/ADS API for my public library (`LIBRARY_ID = "57nm1ouuTMusp62fvjiXYg"`) and writes `data/publications.json`. Fields include bibcodes, authors, journal, DOI, arXiv ID, citation count, abstract, pubdate, and refereed status.

**`update_cv_tex.py`** syncs `data/publications.json` into `assets/cv/JMH_CV.tex`. New entries are inserted at the top of their respective section, and arXiv-only citations are updated to their published journal references once a DOI and volume become available. The script recomputes CV summary statistics (total papers, first-author papers, refereed papers, h-index, total citations) on every run. Entries are categorized by author position: First-Author (position 1), Second-Author (position 2), Co-Major (positions 3–10), and Co-Minor (position 11+). The script is idempotent, detecting existing entries by bibcode and arXiv ID and skipping duplicates.

**`extract_figures.py`** downloads arXiv PDFs for my first-author papers and extracts the first figure from each, storing them at `assets/papers/<arxiv_id>/figures/fig1.png` for display on the publications page.

**`fetch_nasa_image.py`** scrapes the NASA Image of the Day from [nasa.gov/image-of-the-day](https://www.nasa.gov/image-of-the-day/), downloads the featured image to `assets/images/hero_0.{jpg,png}`, and writes `data/nasa_image.json` with the title, caption, credit, and source URL for display on the landing page.

**`generate_og_image.py`** uses Pillow to render a 1200×630 dark-mode social sharing card and saves it to `assets/images/og_image.png`. It draws a subtle star field background, the site's accent color bar, name, job title, affiliation, URL, and a decorative accent circle.

All changes are committed and pushed automatically with the message `chore: daily data update [YYYY-MM-DD]`.

## Publications Page

Publications are fetched from SciX/ADS daily and rendered dynamically from `data/publications.json` by `js/publications.js`. Each card shows the publication month and year (e.g., "Apr 2026"), with first-author papers indicated by a left accent stripe and "First Author" label. Clicking a card or the "Abstract" button expands an inline abstract panel. The "Cite" button copies a formatted BibTeX entry to the clipboard. A search box and filter buttons (All / First Author / Co-Author) are provided. Citation counts are pluralized correctly ("1 citation" vs. "2 citations"). Summary statistics (total papers, total citations, h-index) are displayed for both all papers and published-only papers.

The search box supports ADS/SciX-style field-specific prefixes in addition to free-text search:

| Prefix | Example | Matches |
|--------|---------|---------|
| `first_author:` | `first_author:helton` | Papers whose first author contains the given string |
| `year:` | `year:2025` | Papers published in the given year |

Prefix tokens can be combined with free-text and with each other (e.g., `first_author:helton year:2024`).
