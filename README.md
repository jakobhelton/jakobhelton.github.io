# jakobhelton.github.io

This is the source repository for my personal website at [jakobhelton.github.io](https://jakobhelton.github.io). I am an astrophysicist and Evolving Universe Postdoctoral Fellow in the Department of Astronomy and Astrophysics at The Pennsylvania State University. The site covers my research, publications, presentations, blog, and art.

## Setup

```bash
# 1. Add images to assets/images/:
#    avatar.jpg              — headshot (biography page)
#    hero_1.png              — hero slideshow (landing page)
#    hero_2.png              — hero slideshow (landing page)
#    featured_publication.png — figure for the featured publication card (landing page)
#    og_image.png            — social sharing OG image (auto-generated by the daily pipeline)

# 2. Add art gallery thumbnails (each gallery uses its own featured.jpg):
#    art/painting/featured.jpg
#    art/the_american_midwest_southwest/featured.jpg
#    art/photography_analog/featured.jpg
#    art/photography_digital/featured.jpg
#    art/digital_art/featured.jpg
#    art/home_movies/featured.jpg
#    Note: assets/images/hero_0.jpg is auto-generated by the daily pipeline.

# 3. Place CV at assets/cv/JMH_CV.pdf and assets/cv/JMH_CV.tex

# 4. Add ADS API token as a GitHub secret (ADS_API_TOKEN)
#    Get from: https://ui.adsabs.harvard.edu → Account → API Token

# 5. Enable GitHub Pages (Settings → Pages → Deploy from a branch → main, root)

# 6. Trigger the first data update (Actions → Daily Data Update → Run workflow)
```

## Design

Deep space dark theme (`#08090c` base) with jade green (`#00a878`) accent. Typography follows LaTeX conventions: Playfair Display for headings, Libre Franklin for body text, fully justified with `text-wrap: pretty` for orphan prevention and `text-wrap: balance` on headings. Animated star field (twinkling stars, nebulae, spiral galaxies) runs on every page. Dark and light modes are both supported, with the user's preference persisted in `localStorage`. The layout is fully responsive, with hamburger navigation on mobile.

## SEO

Every page includes a canonical link, `og:image` / `twitter:image` meta tags, and a JSON-LD structured data block (`Person` / `ProfilePage` on biography, `CollectionPage` on publications, `BlogPosting` on blog posts, `WebPage` elsewhere). `sitemap.xml` lists every public URL. When you add new pages (blog posts, art galleries), add a corresponding `<url>` entry to `sitemap.xml`.

## Auto-Updating Pipeline

A GitHub Actions workflow (`.github/workflows/update_data.yml`) runs daily at 06:00 UTC and on manual dispatch. The pipeline executes five scripts in order.

**`fetch_publications.py`** queries the SciX/ADS API for my public library (`LIBRARY_ID = "57nm1ouuTMusp62fvjiXYg"`) and writes `data/publications.json`.

**`update_cv_tex.py`** syncs `data/publications.json` into `assets/cv/JMH_CV.tex`. It recomputes summary statistics (total papers, first-author papers, refereed papers, h-index, total citations) and detects/updates arXiv-only entries once a DOI and volume are available. The script is idempotent.

**`extract_figures.py`** downloads arXiv PDFs for my first-author papers and extracts the first figure from each, storing them at `assets/papers/<arxiv_id>/figures/fig1.png`.

**`fetch_nasa_image.py`** scrapes the NASA Image of the Day, downloads the image to `assets/images/hero_0.jpg`, and writes `data/nasa_image.json` with title, caption, credit, and source URL.

**`generate_og_image.py`** uses Pillow to render a 1200×630 dark-mode social sharing card and saves it to `assets/images/og_image.png`.

All changes are committed and pushed automatically with the message `chore: daily data update [YYYY-MM-DD]`.

## Publications Page

Publications are fetched from SciX/ADS daily and rendered dynamically from `data/publications.json` by `js/publications.js`. Each card shows a status badge (arXiv Preprint / Accepted for Publication / Published in Journal), publication month and year, author position, and citation count. First-author papers have a left accent stripe. Clicking a card expands an inline abstract panel. The "Cite" button copies a BibTeX entry to the clipboard. Summary statistics (total papers, total citations, h-index) are shown for all papers and published-only papers.

The search box supports field-specific prefixes in addition to free-text:

| Prefix | Example | Matches |
|--------|---------|---------|
| `first_author:` | `first_author:helton` | Papers whose first author contains the given string |
| `year:` | `year:2025` | Papers published in the given year |
